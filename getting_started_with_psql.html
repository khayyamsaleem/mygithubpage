Getting Started with PostgreSQL on macOS

1 Installing PostgreSQL

  • brew install postgresql
  • To have a PostgreSQL database server running consistently, so that you can always connect to one for experimentation, you can start it as a "brew service," with brew services start postgres.

2 Setting up a database

When you set up a database, you want a username and password for the "user" that you create to connect to that database. We can create the db and the user to access it with in one command:

  • createdb --username=mike --password sales
  • This creates a table called sales with the username mike and then asks for a password.

3 Creating a table

  • We can create a table with CREATE TABLE statements. For the example I'm dropping in here, you can launch the interpreter and then pass in the CREATE TABLE statement that follows. CREATE TABLE statements can be thought of as initializing your database's schema.
  • psql -U mike sales
    • ask the psql interpreter, with the username mike, to connect to the sales table and prep to receive queries.

  • CREATE TABLE STATEMENT: 

    create table sales(
              cust      varchar(20),
              prod      varchar(20),
              day       integer,
              month     integer,
              year      integer,
              state     char(2),
              quant     integer
)
    • NOTE: The database sales that we defined in the first couple commands differs from the name of the table! A database can have many tables. It is just for this example that the database and the table that we're creating in the database happen to have the exact same name.

4 Populating the table

We populate the table with repeated insert statements. You can run insert statements in the same interpreter that we used before, but what we SHOULD do is write a .sql file that contains all those insert statements for us, so we can re-seed our database should it ever be deleted. Alternatively, we can do it programmatically from a different language like Python.

  • For the sake of this example, you can use a database population script, which can be found here. Once downloaded and placed in the path that you're running the script from, you can use command-line output redirection to send all the files into the query engine.
    • psql -U mike sales < load_sales_table.sql

5 Querying

  • To connect to the database and start running queries, just call psql -U mike sales and try stuff like select * from sales or select count(*) from sales (should be 500). To quit the query interpreter, just type \q.
